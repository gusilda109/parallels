# Sine Sum (float vs double)

Программа заполняет массив из 10⁷ элементов значениями одного полного периода синуса:

​```
data[i] = sin(2π · i / N),   i = 0 … N-1
​```

и считает их сумму. Аналитически сумма равна **0**, но из-за ошибок округления реальный результат от нуля отличается — и тем сильнее, чем меньше точность типа.

## Сборка и запуск

Тип элементов массива (`float` или `double`) выбирается на этапе сборки через опцию CMake `USE_DOUBLE`.

### Float (по умолчанию)

​```
cmake -B build_float -DUSE_DOUBLE=OFF
cmake --build build_float
./build_float/sine_sum
​```

### Double

​```
cmake -B build_double -DUSE_DOUBLE=ON
cmake --build build_double
./build_double/sine_sum
​```

## Результаты

| Тип      | Сумма           |
|----------|-----------------|
| `float`  | `-0.0277862`    |
| `double` | `4.89582e-11`   |

`double` примерно на 9 порядков точнее: при суммировании 10⁷ чисел `float` накапливает ошибку до ~10⁻², тогда как у `double` ошибка остаётся на уровне ~10⁻¹¹.
